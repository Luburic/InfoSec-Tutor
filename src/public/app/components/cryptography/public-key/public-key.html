Public-key cryptography, or asymmetric cryptography, is any cryptographic system that uses pairs of keys: a public key that may be widely distributed, paired with a private key which is available only to the owner. With this we can achieve two things:

  • encryption - by using someone's public key to encrypt a message we can make sure that only the holder of the paired private key can decrypt it, guaranteeing both confidentiality and integrity when paired with a checksum;
  • authentication - by signing a message with our private key anyone can guarantee that the message was created by us which also guaranteeing non-repudation.

The strength of public-key cryptography relies on the degree of computational impracticality for a properly generated private key to be determined from its corresponding public key. In order to achieve this, public-key cryptography systems often rely on cryptographic algorithms based on mathematical problems that currently admit no efficient solution, such as integer factorization and discrete logarithm.

Because of the computational complexity of asymmetric encryption, it is usually used only for small blocks of data, typically the transfer of a symmetric encryption session keys. This symmetric key is then used to encrypt the rest of the potentially long message sequence.
